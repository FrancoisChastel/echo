# references — sites

Personal sites and online publications that hit a register echo wants to occupy. Pull screenshots of each into `docs/moodboard/sites/` in phase 2.

## Writer / artist personal sites

### robinrendle.com

Robin Rendle. CSS-led editorial sensibility. Reads like a printed object. Look at the post pages, not just the index.

### kottke.org

Jason Kottke. Two-decade-old personal blog. Quiet typography, density without noise. Reference for *index page that survives 20 years*.

### craigmod.com

Craig Mod. Essays on walking, books, attention. The reading typography is the masterclass — measure, leading, paragraph rhythm.

### tom-sachs.com

Tom Sachs. Artist site. Big type, hard left-alignment, dark. Different vibe than echo but the *confidence* is worth studying.

### christophniemann.com

Christoph Niemann. Illustrator. Image-led portfolio with zero chrome. Reference for film-page restraint.

### franzheidl.com

Architecture / design portfolio. Editorial grid, minimal motion. Good gradient-of-restraint.

### studio-renee.com / studio-thomas.studio

European design studio sites. Pull both for how studio sites use *one* striking move per page.

## Publications

### are.na

The community side fights the vibe, but the **publication pages** (are.na/about, are.na/annual) are reference-grade.

### lwlies.com (Little White Lies)

Film magazine. Editorial typography online done right. Especially the long reviews.

### its-nice-that.com

Heavier than echo wants to be, but reference for *how a blog page can feel like a print spread*.

### thecreativeindependent.com

Long-form interviews. Wide measure, slow reading typography, low chrome.

## Online journals / portfolios with the right register

### thecityandsoul.com

Diary / log format. Quiet, dated, personal.

### luc.devroye.org

Type designer's reference site. Brutally functional, accidentally beautiful. Useful reference for *the index page nobody designs anymore*.

### ftrain.com (Paul Ford)

Personal essays. Plain typography that ages well. Look at how the navigation almost disappears.

### dwitter.net

Wrong subject matter but right ethos: code-as-design, monospace as primary type, almost no UI.

## Japanese-leaning

### tha.jp (Yugo Nakamura)

Already on the artists list — also a site reference. The whole experience is the work.

### muji.com (especially MUJI HOTEL pages)

Corporate but disciplined. Reference for paper-white surfaces and how silence reads on screen.

### ronnierocket.com

Personal photography journal. Reference for film-grid restraint.

## How to use this list

In phase 2:

1. Capture 1–2 screenshots per site (homepage + one inner page).
2. Annotate each: *what one thing should echo borrow from this?*
3. Build a 2×N grid in `docs/moodboard/README.md` showing each site with its single takeaway.

## Anti-references — explicitly avoid this aesthetic

- Notion-style "minimal" pages with sidebar + cards
- Tailwind + shadcn default landing pages
- Substack and Medium default themes (well-meaning but generic)
- Personal sites that lead with "Hi, I'm X and I do Y" hero blocks
- Dark mode with one accent color called "the design system"
- Any site that uses Inter as both display and body

The fastest way to look like every other 2026 personal site is to use those patterns. We're explicitly rejecting them.
